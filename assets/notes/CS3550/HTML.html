Tags

各种各样的Tags

在HTML中, 文字可以用Tag进行装饰:

This is a text, and <em> this is italic</em> <a href="https://..."> display text</a>

比如上面的<a xxx>xxx</a>和<em>xxx</em>都是Tag - Text Formatting Tag

更多资源

Text Formatting Tags

常见的文本修饰标签有：

  • <a href="...">text</a>
    超链接标签，用来创建跳转 href属性是链接的地址
    <a href="https://example.com">Visit</a>

    不要和<link>搞混了!

  • <em>text</em>
    表示强调，通常渲染成斜体

  • <strong>text</strong>
    表示重点强调，通常渲染成粗体

  • <code>code here</code>
    表示代码片段，通常渲染成等宽字体

  • <img src="..." alt="...">
    插入图片，其中 src 是图片路径，alt 提供文字替代（对无障碍很重要）

Block Tags

用于结构化文本的标签：

<p> paragraph </p>            <!-- 段落 --> 
<h1>Heading 1</h1> ... <h6>Heading 6</h6>   <!-- 六级标题 --> 
<pre> pre formatted text </pre> <!-- 保留空格/换行 -->

这里的<h1>标签和<h>标签不是同样的效果, 后者是用来高亮文字的

Page Structure Tags

用于页面布局的标签：

<header>：页眉区域
<footer>：页脚区域
<main>：页面主体（唯一）
<nav>：导航区域

这些称为 landmark 元素，对无障碍（Accessibility）很重要

Metadata

Tag还可以包含一些数据, 告诉浏览器一些信息:

<!doctype html>           <!-- 声明HTML版本 --> 
<meta charset="UTF-8">    <!-- 网页编码 --> 
<title>Page Title</title> <!-- 页签标题 --> 
<link rel="icon" href="..."> <!-- 网页图标 -->

注意：<head> 中包含这些 Metadata 标签，和 <header> 不同

Lists

<ul>   <!-- 无序列表 -->   
	<li>Item 1</li>   
	<li>Item 2</li> 
</ul>
  
<ol>   <!-- 有序列表 -->   
	<li>First</li>   
	<li>Second</li> 
</ol>

<menu>
	<li>item 1</li>
</menu>

Tables

<table>
	<tr>
        <td>Row1 Col1</td>     
        <td>Row1 Col2</td>   
    </tr>   
    <tr>     
	    <td>Row2 Col1</td>     
	    <td>Row2 Col2</td>   
	</tr> 
</table>
  • <table> 表示表格
  • <tr> 表示一行 table row
  • <td> 表示单元格 table data

Others

Semantics

但为什么这么多不同的Tag呢? 而且就比如<span>和<div>在单独使用后根本就没有区别

首先这些不同名字的tag拥有不同的语义(Semantics), 他们不仅仅可以让不同的开发者一起合作, 更多的还是给css分辨, 这样就可以在我们更改格式的时候不用更改内容

Accessibility

世界上有很多有缺陷的人, 如:

  • Motor, 如帕金森
  • Vision, 如盲人
  • 英语不是第一语言的

让网页增加Accessibility, 不仅仅可以帮助这些有缺陷的人, 还可以让普通人使用起来更加方便

对于盲人来说, 他们会使用阅读器来阅读网页, 而阅读器不像人眼可以看到整个网页, 它是线性的 - 也就是如果我们的网页如果没有结构, 则阅读器需要从开始一直读到想要的地方. 但如果有一个好用的结构, 阅读器可以在<Header>, <nav>和不同的<h1>, <h2>, ...转跳

还有, 就是在网页上, 元素都最好是自洽的, 如 "在这里填入表格", 和"请填入表格". 前者的"这里"很容易被人忽略, 而后者更容易阅读.

在form中的input前, 最好不要使用<p>来描述这个输入框, 而是用<label>, 并在lable中用for="xxx"跟输入框的id进行关联, 这是因为不仅仅阅读器可以在阅读的时候, 会同时读出label和输入框, 不像<p>那样是分开读的
而且如果把输入框放入<label>中还可以让输入框有更大的hitbox (点击label的时候也会自动转跳到输入框), 手机会更容易点击到.

在元素中如<div lang=es>插入语言标识也是一个好处, 比如这里就可以告诉浏览器这部分是西班牙语, 这样浏览器就可以提供翻译等功能

并且文字的阅读性也很重要, 通常需要让contract ratio保持在1:7 或1:4.5左右

总的来说, 要增加网页的Accessibility, 可以遵循:

  • 使用 正确的标题层级 (<h1> → <h2> → <h3>)
  • 为图片添加 alt 描述
  • 在 <html> 中声明语言属性 lang="en" 或 lang="zh"
  • 表单元素要配合 <label> 使用
  • 使用 <main>, <nav>, <header>, <footer> 来标记页面区域

HTML 错误检查

需要能识别和修复常见错误：

  • 标签未闭合
  • 嵌套顺序错误
  • 缺少必要属性（如 <img> 缺 alt）
  • block 和 inline 混用错误
  • 混淆相似标签：
    • <head> vs <header>
    • <a> vs <link>
    • <h1> vs <header>

CSS

在最初的时候, HTML是没有办法进行排版的. 在后面, 增加了一些elements(Tags)可以提供修改格式的功能, 如颜色, 大小等

但这样的问题就是, 每个HTML元素都需要加上同样的Tag来做到格式统一, 尤其是完成整个网页后, 如果想修改某个东西的格式, 是非常困难的 - 需要找到所有的元素然后修改它的Tag

这就是CSS的好处: 另外一个语言专门用来负责表示排版的, 并且一个CSS可以被多个网页共用.

<link rel=stylesheet href="/static/main.css" />

创建和使用CSS

CSS就是一系列的规则, 是由Selectors和body组成的.

strong {color : red;} 
~~~~~~~~~~~~~~~~~~~~~~ ->rule
.action {background : blue;}
        ~~~~~~~~~~~~~~~~~~~->body
.action strong{color : yellow} 
~~~~~~~~~~~~~~~ ->Selector

Selector 就是通过Tag进行选择. 比如我们在没有创建CSS的时候, 浏览器会使用自带的CSS, 这就是为什么<h1>是大字体, 链接是蓝色的.

Selector 作用
header
Any element whose tag name is header
header h1
 Any element whose tag name is h1, inside a header element
#foo
The single element with the id=foo attribute
.action
Any element with a class="action ... attribute
:hover Any element currently being hovered over
body.class 必须同时是class类且是body元素
ul > li 所有作为ul的直接子元素li
h1 + p 所有跟在h1后面的第一个p

上面是一些常用的Selector选择方法

Class和ID都能让Selector定位到某个元素. 但是不同的是同一个Class可以对多个元素进行标记, 而一个网页中只能有一个相同的ID
在元素上挂上class=就可以标注某个元素是什么class了

<h2>Cat: <em class="class feature">ABC<em/><h2/>

比如上面的例子, 就是文字ABC拥有了两个Class, 类名分别是class和feature

a:hover{
 color : yellow
}

上面这种连一起的就是两种Selector都要满足: 这样鼠标悬浮在链接上, 链接就变成了黄色

有时候, 我们写的CSS可能没有效果, 这是因为可能有些CSS被覆盖掉了. 我们可以在浏览器中的Style中看到CSS的优先级: 如果有CSS的优先级太低, 则他会在Debugger中被划掉.
这个优先级的评判标准是按照它修改了几个ID, Class, Tag表示的. 修改的数量越多, 则它的优先级越高.

想在页面中使用CSS, 可以使用link去使用CSS

<link rel=stylesheet href="./style.css">

CSS的继承性质

当我们把Selector选择成html, 则表示我们选择的是网页端根元素 - 这么修改完以后, 所有的元素默认的格式都会是这个rule的body

常见的CSS属性

Text

	element.style{
		font-size: 24px;
		font-weight:700;
		font-family:"字体名";
		font-style:italic; 
		text-decoration:underline; 
		text-align: justify;
		color: blue;
	}

Color

#alert {color : #a0111f;}
#alert {color : navy;}

Border

如果想给物体加上边框, 则我们需要定义三个属性: 边框粗细, 边框样式, 边框颜色

.feature{
	border: 5px, solid, #000000;
	border-radius: 3px;
}

.feature{
	border-top: 1px, dotted, #000000;
}

.feature{
	border-left: 1px, dashed, #000000;
}

.feature{
	border-top-width: 1px;
	border-top-style: solid;
	border-top-color: #000000;
}

.feature{
	border-width: 10px, 0, 10px, 0; //上左下右的边框宽度 
}
